Title: Multi-Agent Specialized Team (Solo Founder Setup)

## Summary

This source describes running multiple specialized OpenClaw agents as a coordinated team (e.g., strategy, business, marketing, dev), each with distinct role/personality/model, shared memory docs, and scheduled tasks.

Key ideas:
- Role specialization beats one overloaded general agent
- Shared memory + private context per role
- Single chat control plane with role-based routing
- Scheduled proactive tasks create compounding value
- Start with a small team and expand gradually
